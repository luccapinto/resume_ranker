## Contexto
Este é um projeto novo em monorepo. Para desenvolver localmente e rodar a suíte inteira de forma determinística, precisamos de um ambiente local configurado com Docker Compose que gerencie o banco relacional (PostgreSQL) e o banco vetorial (Qdrant).

## Objetivo
Configurar o ambiente Docker Compose com PostgreSQL e Qdrant, estruturar o monorepo dividindo em `/api` (backend FastAPI com Python 3.11+) e `/web` (frontend Next.js com TypeScript e Node.js 18+), e garantir scripts para rodar os testes e verificar liveness.

## Especificação Técnica
- **Infraestrutura (Docker Compose)**:
  - Serviço `postgres`: imagem oficial `postgres:15-alpine` com volumes persistentes locais e credenciais expostas via variáveis de ambiente.
  - Serviço `qdrant`: imagem oficial `qdrant/qdrant:latest` com persistência em volume e porta `6333` exposta.
  - O docker-compose deve expor as portas adequadas para desenvolvimento local (`5432` para Postgres, `6333` para Qdrant).
- **Backend (FastAPI)**:
  - Esqueleto básico sob `/api` com `main.py` expondo endpoints de healthcheck (`/health`).
  - Utilizar Poetry ou Pipenv para gerenciar dependências. Configurar arquivos essenciais: `pyproject.toml`, `requirements.txt` ou `Pipfile`.
  - Configurar biblioteca de ORM (SQLAlchemy ou SQLModel) apontando para o Postgres do compose.
- **Frontend (Next.js)**:
  - Esqueleto básico sob `/web` inicializado com Next.js (Page Router ou App Router) utilizando Tailwind e TypeScript.
  - Adicionar script de lint e build básico para validação.
- **Root**:
  - Script no root ou Makefile para inicializar o ambiente com um único comando (`docker-compose up -d`).

## Critérios de Aceite
- [x] Rodar `docker-compose up -d` inicializa Postgres e Qdrant sem erros.
- [x] O backend `/api` consegue se conectar ao Postgres e Qdrant utilizando variáveis de ambiente de desenvolvimento.
- [x] Endpoint `/api/health` retorna `{"status": "ok", "postgres": "connected", "qdrant": "connected"}`.
- [x] A aplicação frontend `/web` inicializa localmente com `npm run dev` ou similar sem erros de compilação.
- [x] O backend gera o arquivo schema OpenAPI no startup ou via script para uso futuro na tipagem do frontend.
