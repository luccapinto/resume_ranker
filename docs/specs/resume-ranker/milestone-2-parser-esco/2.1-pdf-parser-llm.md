# Card 2.1 — Parser de PDF e Extração Estruturada via LLM

## Contexto
Tanto os currículos (normalmente enviados em formato PDF) quanto as descrições de vagas (texto ou documento) precisam ser normalizados para o mesmo schema estruturado antes de qualquer representação vetorial. Isso assegura que estejamos comparando maçãs com maçãs no motor de busca e não textos brutos desorganizados.

## Objetivo
Implementar o pipeline de leitura de PDFs de currículos usando PyMuPDF, extrair o texto limpo, enviar ao redator de PII (Card 1.2) e, em seguida, utilizar uma chamada estruturada de LLM via OpenRouter para extrair os dados normalizados no schema unificado definido com Pydantic.

## Especificação Técnica
- **Parser de PDF**:
  - Utilizar a biblioteca PyMuPDF (`fitz`) para ler o texto dos PDFs enviados.
  - Limpar caracteres especiais ou quebras de linha excessivas.
- **Esquema de Dados Unificado (Pydantic)**:
  - Criar um schema base compartilhado para perfis: `BaseProfile`.
  - Definir os modelos: `CandidateProfile` e `JobRequirements` herdando do base ou contendo os mesmos campos fundamentais:
    - `seniority` (Enum: `Estágio`, `Júnior`, `Pleno`, `Sênior`, `Especialista/Lead`).
    - `skills_raw` (Lista de strings das competências extraídas, ex: `['Python', 'React', 'Git']`).
    - `experience_years` (Float: Anos totais de experiência inferidos).
    - `education` (Lista de objetos contendo grau, área e ano).
    - `certifications` (Lista de strings contendo certificações extraídas).
    - `languages` (Lista de strings de idiomas dominados).
    - `narrative_experience` (Texto consolidado descrevendo a trajetória profissional e responsabilidades — este texto será usado para o `narrative_vector`).
- **Extração com LLM (OpenRouter)**:
  - Configurar um cliente para API do OpenRouter.
  - Implementar chamadas utilizando a funcionalidade de Structured Outputs (JSON Schema/Pydantic validation).
  - O prompt do LLM deve receber o texto **já anonimizado** pelo `PIIRedactor` e retornar o JSON correspondente ao schema Pydantic.
  - Validar e instanciar os dados retornados utilizando o validador do Pydantic.
- **Persistência**:
  - Salvar no PostgreSQL o texto original limpo (para explicabilidade e auditoria), o texto anonimizado e o JSON final do perfil extraído.

## Critérios de Aceite
- [ ] O parser extrai texto legível de PDFs complexos de currículos (colunas duplas, tabelas simples, layouts modernos).
- [ ] O backend valida que currículos e vagas compartilham a mesma estrutura interna no banco de dados.
- [ ] O LLM extrai com sucesso o perfil estruturado seguindo o schema Pydantic, convertendo campos informais para o enum de senioridade correto e computando os anos de experiência.
- [ ] Tratamento adequado de erros caso o JSON retornado pelo LLM seja inválido (retry automático ou log legível para intervenção humana).

### Testes Automatizados
