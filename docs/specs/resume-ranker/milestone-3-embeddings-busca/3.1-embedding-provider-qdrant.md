# Card 3.1 — Abstração de `EmbeddingProvider` e Ingestão Qdrant

Tag: 🤖 Execução Autônoma

## Contexto
O matching bidirecional candidato-vaga utiliza busca semântica em duas dimensões principais: competências técnicas (normalizadas pela ESCO) e narrativa de experiência (texto livre da trajetória). Para isso, as coleções de dados do Qdrant precisam ter vetores nomeados correspondentes e aceitar tanto embeddings locais quanto externos por configuração de ambiente.

## Objetivo
Implementar a interface de `EmbeddingProvider` para gerar representações vetoriais densas e esparsas, configurar as coleções `candidates` e `jobs` no Qdrant com suporte a múltiplos vetores por ponto e persistir os payloads estruturados de forma otimizada para filtragem determinística.

## Especificação Técnica
- **Interface `EmbeddingProvider`**:
  - Definir a interface abstrata contendo métodos `get_dense_embedding(text: str) -> List[float]` e `get_sparse_embedding(text: str) -> Dict[int, float]` (ou formato de vetor esparso nativo do Qdrant).
  - Implementação Local: Utilizar `sentence-transformers` com um modelo multilíngue do leaderboard MTEB (ex: `paraphrase-multilingual-MiniLM-L12-v2` ou `multilingual-e5-small/large`).
  - Implementação Remota: Provedor de API configurável (OpenAI `text-embedding-3-small/large` ou Voyage AI).
- **Esquema de Vetores no Qdrant**:
  - Criar/configurar as duas coleções: `candidates` e `jobs`.
  - Cada ponto nas coleções deve conter:
    - Vetor nomeado `skills_vector`: embedding gerado a partir da concatenação de labels das competências ESCO mapeadas.
    - Vetor nomeado `narrative_vector`: embedding gerado a partir do texto de narrativa de trajetória extraído.
    - Vetor esparso nomeado `lexical_vector`: gerado através de um tokenizador BM25/Splade local para casar termos raros exatos.
- **Payload Estruturado**:
  - Armazenar metadados no payload de cada ponto no Qdrant:
    - `esco_skills_ids`: lista de URIs/IDs canônicos da ESCO.
    - `seniority`: string com o nível inferido.
    - `experience_years`: float.
    - `certifications`: lista de certificações.
    - `candidate_id` / `job_id` vinculados à tabela do PostgreSQL.

## Critérios de Aceite
- [ ] Coleções no Qdrant são criadas no startup se não existirem, com as configurações corretas de dimensões vetoriais por nome.
- [ ] O `EmbeddingProvider` alterna corretamente entre o modelo local e API externa via variável de ambiente `EMBEDDING_PROVIDER`.
- [ ] O pipeline gera embeddings de forma simétrica para perfis de candidatos e requisitos de vagas.
- [ ] Suporte a vetores esparsos integrado na inserção de dados do Qdrant sem lançar erros de formato.
