# Card 3.2 — Busca Híbrida Bidirecional e Reranking com Cross-Encoder
Status: [x] Concluído <!-- Opções: [ ] Pendente, [/] Em Progresso, [x] Concluído -->
Tag: 🤖 Execução Autônoma

## Contexto
Para obter o melhor dos mundos lexical, semântico e estruturado, a busca do Resume Ranker não pode depender apenas de similaridade de cosseno simples. Ela deve mesclar correspondência exata de termos (vetor esparso), similaridade de conceitos/narrativa (vetores densos nomeados) e regras de negócios rígidas (filtros de senioridade e requisitos obrigatórios), seguidos por um reranking semântico profundo.

## Objetivo
Implementar a engine de busca híbrida bidirecional que consulta a coleção oposta (candidato -> vaga ou vaga -> candidato) no Qdrant combinando filtros estruturados nativos com busca vetorial denso/esparso unida por Reciprocal Rank Fusion (RRF), e refinar o top-K resultante utilizando um modelo Cross-Encoder local.

## Especificação Técnica
- **Combinação de Filtros e Busca Híbrida**:
  - Utilizar a API de busca do Qdrant (com `SearchRequest` ou multi-search queries).
  - Aplicar filtros estruturados determinísticos em tempo de busca (ex: filtrar candidatos que possuam anos mínimos de experiência ou certificações obrigatórias).
  - Executar a busca de similaridade denso+esparso combinando pontuações de `skills_vector`, `narrative_vector` e `lexical_vector`.
- **Reciprocal Rank Fusion (RRF)**:
  - Implementar o algoritmo de RRF no backend para fundir as listas ranqueadas geradas pelas diferentes buscas vetoriais (`skills`, `narrative` e `lexical`).
  - Utilizar a fórmula padrão de RRF:
    \[
    RRF\_Score(d) = \sum_{m \in M} \frac{1}{k + r_m(d)}
    \]
    onde \(M\) é o conjunto de métodos de busca (denso-skills, denso-narrativa, esparso-lexical), \(r_m(d)\) é a posição do documento \(d\) no ranking do método \(m\), e \(k\) é uma constante de suavização (padrão \(k=60\)).
- **Reranker com Cross-Encoder**:
  - Carregar um modelo Cross-Encoder local leve (ex: `cross-encoder/ms-marco-MiniLM-L-6-v2` via `sentence-transformers`).
  - Enviar o texto de consulta e os perfis do top-K (gerados na busca híbrida) para o Cross-Encoder estimar a pontuação de relevância de forma cruzada (relação direta entre query e documento).
  - Ordenar a lista final com base no score do Cross-Encoder obtendo o top-N para presentation.

## Critérios de Aceite
- [x] A API de busca retorna resultados com score híbrido e ordenação refinada pelo Reranker.
- [x] O filtro determinístico nativo do Qdrant elimina corretamente os registros que não cumprem os pré-requisitos estruturados (ex: certificação específica obrigatória).
- [x] O RRF funde as notas sem causar overflows ou inversões erráticas de relevância.
- [x] O Cross-Encoder executa o reranking em menos de 800ms para um top-K de 20 elementos locais.
