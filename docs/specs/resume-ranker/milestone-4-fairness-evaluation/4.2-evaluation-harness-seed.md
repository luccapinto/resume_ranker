# Card 4.2 — Harness de Avaliação de Retrieval e Seed Data

## Contexto
Decisões de engenharia em motores de busca (como o peso relativo de `skills_vector` vs. `narrative_vector` vs. vetor esparso) não devem ser baseadas em achismo. Precisamos de um Harness de Avaliação quantitativo que compare empiricamente diferentes parametrizações usando métricas consolidadas de Information Retrieval.

## Objetivo
Implementar o Harness de Avaliação que calcula métricas de NDCG e MRR contra um dataset rotulado, e carregar o Seed Data sintético rico (50 candidatos e 10 vagas, com casos de borda e pares adversariais) para povoar o sistema inicial.

## Especificação Técnica
- **Dataset de Relevância Rotulado (Qrels)**:
  - Definir um arquivo de gabarito (relevance judgments, ex: formato TREC Qrels ou JSON estruturado) que mapeia pares de (ID da Vaga, ID do Candidato) para notas de relevância de 0 a 3 (0: irrelevante, 1: pouca relevância, 2: relevante, 3: altamente relevante).
- **Harness de Avaliação**:
  - Implementar scripts no backend que executam consultas automáticas para as vagas cadastradas no dataset.
  - Computar as métricas:
    - **NDCG@K** (Normalized Discounted Cumulative Gain): Para avaliar a qualidade do ordenamento no topo do ranking.
    - **MRR** (Mean Reciprocal Rank): Para avaliar a velocidade com que o primeiro resultado relevante é encontrado.
  - Permitir a execução de benchmarks comparando diferentes pesos (ex: 70% skills / 30% narrativa vs. 50%/50%) e exportar um relatório comparativo estruturado.
- **Carga de Seed Data**:
  - Script Python para carregar no banco e no Qdrant:
    - **50 Perfis de Candidatos Sintéticos**: Cobrindo diversidade de senioridade, transições de carreira (ex: ex-designer migrando para front-end), falsos-positivos propositais (mesmo cargo, mas sem a certificação obrigatória listada na vaga) e lacunas de emprego (career gaps).
    - **10 Vagas Sintéticas**: Com descrições claras de requisitos técnicos, formação e certificações duras.
    - **Par Adversarial (Módulo de Fairness)**: 20 currículos adicionais (10 pares) focados no teste de equidade para o Card 4.1.

## Critérios de Aceite
- [ ] O script de seed data insere com sucesso todos os 70 currículos e 10 vagas no Postgres e gera seus respectivos vetores no Qdrant.
- [ ] O script do Harness calcula NDCG@5, NDCG@10 e MRR com precisão matemática.
- [ ] O Harness de Avaliação roda por linha de comando (`python -m api.eval.run_harness`) e imprime uma tabela comparativa comparando pelo menos 3 configurações de pesos vetoriais diferentes.

### Testes Automatizados
- [ ] Teste unitário para as fórmulas de NDCG e MRR com arrays de relevância fixos e controlados.
- [ ] Validar que o seed data é carregado de forma determinística em bancos limpos durante a execução de testes de integração.
