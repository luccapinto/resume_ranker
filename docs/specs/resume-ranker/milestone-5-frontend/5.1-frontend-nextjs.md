## Contexto
O usuário final (recrutadores ou gestores) precisa interagir com o motor de busca bidirecional, visualizar o ranqueamento, auditar o viés e ver as justificativas de forma transparente. Uma interface web moderna, fluida e com design premium é fundamental para materializar esses conceitos de arquitetura aberta.

## Objetivo
Desenvolver o frontend web do Resume Ranker sob a pasta `/web/` usando Next.js, Tailwind CSS e Shadcn/ui, garantindo integração de tipos TypeScript gerados automaticamente a partir do OpenAPI do FastAPI.

## Especificação Técnica
- **Aparência e Design Premium**:
  - Tema escuro padrão (Dark Mode) com elements inspirados em Glassmorphism (efeitos de desfoque de fundo, bordas translúcidas finas, gradientes sutis e cores HSL harmônicas).
  - Uso de tipografia premium (Inter ou Outfit) carregada via Google Fonts.
  - Micro-animações em transições de tabelas, expansão de cards e interações de botões usando Framer Motion ou transições Tailwind nativas.
- **Telas Principais**:
  - **Tela de Busca & Ranking**:
    - Seletor bidirecional (Toggle: "Buscar Candidatos para uma Vaga" vs. "Buscar Vagas para um Candidato").
    - Campo de upload de PDF (currículo) ou campo de texto para vaga com feedback visual de progresso (ingestão assíncrona).
    - Tabela de resultados contendo: Score de Similaridade (denso/esparso), Score de Reranking (Cross-Encoder) e Botão "Ver Justificativa".
    - Painel expansível (Accordion ou Modal) de Explicabilidade com o texto explicativo gerado por LLM e destaques visuais (tooltips ou badges) das citações validadas contra o texto original.
  - **Tela de Perfil Estruturado**:
    - Comparador lado-a-lado: Perfil estruturado extraído (`CandidateProfile` ou `JobRequirements`) com as competências mapeadas para ESCO (com badges indicando a URI e relação hierárquica oficial) vs. o Texto Original completo.
  - **Painel de Auditoria de Fairness**:
    - Visualização dos logs de auditoria do banco de dados.
    - Status do último teste contraditório de viés, com gráficos ou cards estatísticos exibindo a variação média de score entre gêneros, nomes e etnias artificiais (garantindo transparência na neutralidade do sistema).
- **Integração Técnica**:
  - Utilizar a biblioteca `openapi-typescript` ou similar para gerar automaticamente os arquivos de tipos TypeScript com base no endpoint `/openapi.json` da API FastAPI.
  - Configurar chamadas de API utilizando React Query (TanStack Query) ou hooks nativos eficientes para gerenciar estados de loading e revalidação.

## Critérios de Aceite
- [x] O frontend compila sem erros de TypeScript usando os tipos OpenAPI auto-gerados.
- [x] A busca bidirecional funciona de ponta a ponta: ao selecionar uma vaga, o ranking de candidatos com scores e explicações é exibido sem recarregar a página inteira.
- [x] O design segue fielmente a paleta de cores escura e estética translúcida (Glassmorphism), sendo totalmente responsivo para diferentes tamanhos de tela.
- [x] Painel de explicabilidade exibe apenas as citações reais e destaca trechos que batem com o texto original do candidato/vaga.
